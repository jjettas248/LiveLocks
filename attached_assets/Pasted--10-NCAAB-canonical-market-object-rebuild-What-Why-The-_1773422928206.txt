#10 - NCAAB canonical market object rebuild
What & Why
The NCAAB engine currently emits markets as a flat array (marketVerdicts) and the UI reconstructs market data from a mix of array lookups, legacy play-level fields, and silent fallbacks. This produces incorrect probabilities, mislabeled lines, and inconsistent behaviour across the Full Game / H1 / H2 tabs.

The fix is to rebuild the engine/UI contract from scratch around a single canonical typed market object: engineOutput.markets. The engine computes markets once. The UI renders them directly. No legacy fields. No UI inference. No silent fallbacks.

Done looks like
engineOutput.markets is a keyed object with exactly six keys (full_total, full_spread, h1_total, h1_spread, h2_total, h2_spread). Each key is always present; available: false when no real sportsbook line exists or the game state does not permit the market.
Game state gating is enforced in the engine: H1 markets flip to available: false once halftime begins; H2 markets remain available: false until the engine has both a valid halftime state and a sportsbook H2 line.
projection is never rendered as a sportsbook line anywhere in the UI.
modelProb is clamped to [1, 99] in the engine. If calibration produces NaN, Infinity, or undefined, that market is set to available: false and not rendered.
edge is computed once in the engine as modelProb − bookImpliedProb; the UI reads it directly and never recomputes it.
confidenceTier is assigned by the engine (ELITE / STRONG / VALUE / NONE) based on edge; the UI never recomputes it.
A market with available: false renders as "Unavailable" — no substitution, no fallback to engine projections or full-game values.
H1 tab shows only H1 market data; H2 tab shows only H2 market data.
marketVerdicts array is derived from markets (not maintained independently) for backward compatibility with any non-UI consumers.
A "Top Plays" feed aggregates all available === true markets across all games, sorts by Math.abs(edge) descending, and renders the top 20 results maximum.
Each market card has an "Add to Bet Card" action that passes the canonical NCAABMarket object.
play.overProb, play.spreadProb, play.h2EngineOverProb, play.h2SpreadLine, marketVerdicts.find(), inline toFixed probability strings, ?? 50 fallbacks, and ?? overProb fallbacks are gone from all UI render paths.
Existing data flow for NBA and MLB is untouched.
Safety guardrails (mandatory)
Guardrail 1 — Game state market gating

full_total and full_spread: available pre-game and during live play.
h1_total and h1_spread: available pre-game and during the first half only; must be available: false once halftime begins.
h2_total and h2_spread: available: false until halftime has started AND a sportsbook H2 line exists. Never available pre-game or during the first half.
Guardrail 2 — Probability safety

modelProb clamped to [1, 99] before assignment. If calibration produces NaN, Infinity, or undefined, set available: false for that market.
The UI must never display probabilities outside the [1%, 99%] range.
Guardrail 3 — Projection vs book line separation

projection is an internal model value only.
bookLine must always originate from a sportsbook feed. No sportsbook line → available: false.
Guardrail 4 — Helper function deprecation safety

Mark resolveSpreadVerdict and resolveVerdictForMarket with a @deprecated comment before touching them.
Run a project-wide grep to confirm they are unused outside ncaab-admin-tab.tsx.
Only delete them if no other module references them.
Guardrail 5 — Top Plays performance limit

Cap the Top Plays feed at the top 20 markets globally to prevent excessive DOM rendering on large slates.
Guardrail 6 — Confidence tier generation

The engine assigns confidenceTier during market construction using edge thresholds: edge ≥ 12 → ELITE, edge ≥ 8 → STRONG, edge ≥ 4 → VALUE, else → NONE.
The UI reads confidenceTier directly; it does not recalculate it.
Guardrail 7 — Sportsbook odds validation

Odds must fall within [-2000, +2000]. Odds equal to 0, null, undefined, or outside this range invalidate the market: set available: false and do not compute bookImpliedProb or edge. This prevents corrupted sportsbook feeds from generating false edges.
Guardrail 8 — Engine debug markets (non-production only)

When NODE_ENV !== "production", expose an optional debugMarkets array on engineOutput. Each entry contains: marketKey, projection, bookLine, modelProb, bookImpliedProb, edge. This structure is for diagnostics only and must never drive UI rendering.
Guardrail 9 — Top Plays deduplication

Allow only one entry per marketKey per game in the Top Plays feed. If multiple sides of the same market exist (e.g. OVER and UNDER), keep the side with the higher absolute edge. After deduplication, sort by Math.abs(edge) descending and cap at 20 entries.

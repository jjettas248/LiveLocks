#27 - MLB: PA Distribution Model (Lineup Turnover)
What & Why
Upgrade the MLB hits probability engine by replacing the deterministic remaining plate appearance (PA) estimate with a probabilistic PA distribution model. Instead of a single averaged remainingPA value, the engine calculates a probability distribution across possible remaining PAs (1, 2, or 3). The final hit probability becomes a distribution-weighted binomial result.

The binomial threshold fix (Math.ceil(bookLine) and neededHits === 0 short-circuit) was already merged in Task #25. Do NOT re-apply or modify that logic.

Done looks like
server/mlb/paDistribution.ts exists and exports estimatePADistribution() with a local clamp helper, correct slot adjustments, zero-sum guard, and normalized output { 1: number, 2: number, 3: number }
calculateHitsEdge() in markets.ts uses estimatePADistribution and replaces the single binomial call with a distribution-weighted loop
expectedHits is the distribution-weighted sum: paDist[1] * adjustedRate + paDist[2] * 2 * adjustedRate + paDist[3] * 3 * adjustedRate
remainingPA on the output is the weighted average (1 * paDist[1]) + (2 * paDist[2]) + (3 * paDist[3]) — kept as a float, not rounded
output.paDistribution is set to paDist

Tasks
Create server/mlb/paDistribution.ts — Implement exactly as follows:

Local helper: const clamp = (v: number, min: number, max: number) => Math.min(max, Math.max(min, v))
paceFactor = clamp(currentRuns / leagueAvgRuns, 0.85, 1.15)
Slot adjustments (default -0.15): slot ≤ 2 → 0.20, slot ≤ 5 → 0.10, slot ≤ 7 → -0.05
Raw values: pa1 = clamp(0.35 + slotAdj + (inning * -0.02), 0, 1), pa2 = clamp(0.45 + paceFactor * 0.05, 0, 1), pa3 = clamp(0.20 + (9 - inning) * 0.02, 0, 1)
Zero-sum guard: if pa1 + pa2 + pa3 === 0, return { 1: 0.34, 2: 0.33, 3: 0.33 } immediately
Otherwise normalize: pa1 /= sum; pa2 /= sum; pa3 /= sum
Return { 1: pa1, 2: pa2, 3: pa3 }
Update calculateHitsEdge() in markets.ts — Import estimatePADistribution from ./paDistribution. After computing adjustedRate, call estimatePADistribution(input.inning, input.lineup.battingOrderSlot, input.currentRuns ?? 4.5, input.leagueAvgRuns ?? 4.5) to get paDist. Replace the single binomial call with a loop over Object.entries(paDist), accumulating rawProbabilityOver += binomialOverProbability(paCount, adjustedRate, neededHits) * paProb (signature: remainingPA, adjustedHitRate, neededHits). Compute expectedHits = paDist[1] * adjustedRate + paDist[2] * 2 * adjustedRate + paDist[3] * 3 * adjustedRate. Set remainingPA to (1 * paDist[1]) + (2 * paDist[2]) + (3 * paDist[3]) — do not round. Set output.paDistribution = paDist.

Update MLBPropOutput in types.ts — Add optional field paDistribution?: { 1: number; 2: number; 3: number }. Do not modify any existing fields.

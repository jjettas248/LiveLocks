#28 - Fix MLB Live UI State & Interaction Parity

Goal
Repair MLB Live UI rendering and interaction bugs so the page behaves
identically to NBA Live.

The MLB Live page currently suffers from:
• game header collapsing to "@ — Box Score"
• incorrect render order
• player rows not triggering calculator mode
• signal cards visually broken (probability ring overlap)
• calculator and signals rendering simultaneously
• UI instability during polling refreshes

System philosophy (unchanged)

Default state:
Today's Games
↓
Inning Tabs
↓
Edge Signals
↓
Box Score

Interaction:
Click player row
↓
Calculator Mode
↓
Back button returns to signals + box score

Polling behavior must remain active even when signals are hidden.

<TodaysGames />
<InningTabs />
<EdgeSignals />

calculator mode is active.

Wrap both sections:

<div style={{ display: selectedPlayer ? "none" : "block" }}>
  <EdgeSignals />

  <BoxScore />

2. Stabilize game header

Polling refetches temporarily unset selectedGame which causes the
header to collapse.

Create stable reference:

const selectedGameRef = useRef<Game | null>(null)

useEffect(() => {
  if (selectedGame) {
    selectedGameRef.current = selectedGame
  }
}, [selectedGame])

Use fallback:

const game = selectedGame || selectedGameRef.current

Render header:

<h2 className="text-lg font-semibold">
  {game?.awayTeam} @ {game?.homeTeam}
</h2>

The box score toggle should only control the table visibility.

------------------------------------------------

3. Fix box score collapse toggle

Add state:

const [boxExpanded, setBoxExpanded] = useState(true)

Toggle:

<button
 onClick={() => setBoxExpanded(prev => !prev)}
 className="flex items-center gap-2 cursor-pointer"
>

{boxExpanded ? "▾ Box Score" : "▸ Box Score"}

</button>

Render table:

{boxExpanded && (
  <BoxScoreTable />
)}

4. Make player rows clickable

<tr

 className="cursor-pointer hover:bg-neutral-800 transition"

5. Fix calculator mode isolation

When a player is selected only calculator content should render.

Add condition:

{selectedPlayer && (
  <div>

    <button
      onClick={() => setSelectedPlayer(null)}
      className="mb-4"
    >
      ← Back to Game
    </button>

    <MatchupDetailsPanel player={selectedPlayer} />

    <LiveLineSelector player={selectedPlayer} />

    <PredictionResultPanel player={selectedPlayer} />

  </div>
)}

Signals and box score remain mounted but hidden via display:none.

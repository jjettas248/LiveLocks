#3 - NCAAB Engine End-to-End Rebuild

What & Why
The NCAAB live engine has three related problems:
1. Projection math sometimes diverges from what gets displayed
2. Pick direction (Over/Under) can contradict the actual projection
3. Probabilities lack calibration and sometimes produce inflated 80–90% edges that do not reflect reality

The goal is to rebuild the engine in a modular, auditable way so that every displayed value traces directly back to the underlying calculation, with no silent contradictions.

Important implementation rule
Treat this task as a server-side engine integrity rebuild first.

Do NOT:
- redesign UI
- add non-essential abstractions
- introduce client-side recomputation

until the following conditions are true:

1. projection math is correct
2. probabilities are calibrated
3. pick direction is consistent
4. output validation tests pass

--------------------------------------------------

Done looks like

- The displayed projection, probability, and pick direction are always mathematically consistent
- Over is never recommended when the projection is clearly below the line (and vice versa)
- Probabilities rarely exceed realistic thresholds for NCAAB totals/spreads
- Full game total, 1H total, 2H total, spread, team total, 1H team total, and 2H team total all use appropriate game-level accumulation logic
- Every settled play logs the full diagnostic chain:
  line
  projection
  final result
  displayed side
  raw probability
  calibrated probability
- Validation function catches contradictions at output time
- Client renders only from final typed engine output returned by server
- Test cases covering contradiction scenarios pass

--------------------------------------------------

Out of scope

- Changing data sources (ESPN, SGO, Odds API, Torvik)
- NCAAB player prop lines (this engine is game-level totals/spreads/team totals only)
- Probability calibration foundation already built in Task #1
- Redesigning NBA tab

--------------------------------------------------

5. Rebuild probability layer

Create:

calculateNCAABProbabilities(input, projectionResult)

Replace linear formula:

50 + diff * mult * 0.3

Use realistic distribution model.

Variance model:

NCAAB scoring variance  
σ = 12 points (default)

Allow overrides later but start deterministic.

Return:

rawOverProb  
rawUnderProb  
rawSpreadProb  
raw1HProb  
raw2HProb  

Then apply calibration:

calibrateNCAABProbability(rawProbability, marketType, context)

Guardrails:

- probabilities rarely exceed 75–80%
- if above threshold → emit warning to debug payload

--------------------------------------------------

6. Dynamic multiplier guardrail

Clamp dynamic multiplier:

0.6 ≤ multiplier ≤ 1.4

Log warning if attempted value exceeds bounds.

--------------------------------------------------

7. Deterministic edge rule

Recommended side must only be returned if BOTH conditions are met:

absolute projection gap ≥ 2.0 points  
calibrated probability ≥ 57%

Otherwise return:

NO_EDGE

--------------------------------------------------

8. Fix pick-direction logic

Create function:

determineRecommendedSide(projectedValue, line, overProb, underProb, edgeThreshold)

Must:

- compare projection vs line
- compare calibrated probabilities
- reject contradictions

Contradictions to detect:

projection below line but side = OVER  
projection above line but side = UNDER  
tiny edge but side returned  
probability mismatch with displayed side  

Allowed outputs:

OVER  
UNDER  
NO_EDGE

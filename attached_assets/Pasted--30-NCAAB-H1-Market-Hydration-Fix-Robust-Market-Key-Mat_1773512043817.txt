#30 - NCAAB H1 Market Hydration Fix (Robust Market Key Matching)

What & Why
Sportsbooks publish live first-half totals and spreads, but the engine frequently shows "1st Half lines unavailable" because the ingestion layer relies on exact string matches for sportsbook market keys.

Different sportsbooks return slightly different identifiers such as:

h1_totals
1h_totals
firsthalf_totals
first_half_totals
1st_half_totals
h1_spreads
1h_spreads
firsthalf_spreads

Exact matches cause the extraction logic to silently fail.

The engine must detect these markets using tolerant normalized matching and must never fabricate synthetic first-half lines.

Done looks like
When SGO or The Odds API provides live H1 totals or spreads, the engine receives non-null h1TotalLine and h1SpreadLine values.
H1 TOTAL and H1 SPREAD rows populate with real sportsbook lines.
If sportsbooks genuinely do not provide H1 markets, the engine returns unavailableMarket().
In non-production environments, logs list all detected sportsbook market keys.
If H1-like keys exist but extraction fails, a hydration warning fires.

Out of scope
Synthetic market generation
Changes to CLV logic
Changes to public fade logic
Changes to projections
UI changes

Tasks

Create shared market key normalization helper

Add helper:

function normalizeMarketKey(key: string) {
  return key
    .toLowerCase()
    .replace(/[_\-]/g, "")
    .replace(/\s+/g, "")
}

Example:

"FIRST_HALF_TOTALS"
→ "firsthalftotals"

Use token-based matching

Create helper:

function isH1TotalKey(key: string) {
  const k = normalizeMarketKey(key)
  return k.includes("1h") || k.includes("firsthalf") || k.includes("1sthalf")
}

function isH1SpreadKey(key: string) {
  const k = normalizeMarketKey(key)
  return (
    k.includes("1hspread") ||
    k.includes("firsthalfspread") ||
    k.includes("1sthalfspread")
  )
}

Update SGO extraction

In extractSGO1HLines:

Instead of:

odds["points-all-1h-ou-over"]

Search through all keys and select those where:

isH1TotalKey(key)
or
isH1SpreadKey(key)

Log matched keys in debug mode.

Update Odds API extraction

Replace exact matches:

bk.markets.find(m => m.key === "h1_totals")

with tolerant matching:

bk.markets.find(m => isH1TotalKey(m.key))

Do the same for spreads.

Hydration diagnostics

After SGO + Odds API extraction:

If h1TotalLine or h1SpreadLine is null:

Check if any sportsbook market key contains:

"1h"
"half"
"1st"

If yes emit:

console.warn("H1 market hydration failure", {
  gameId,
  sportsbookMarkets
})

In non-production always log:

console.debug("Detected sportsbook markets", {
  gameId,
  markets: sportsbookMarketKeys
})

Remove synthetic fallback

Replace:

rawH1TotalLine !== null
  ? rawH1TotalLine
  : total !== null
    ? Math.round(total * NCAAB_H1_FRACTION * 2) / 2
    : null

with:

rawH1TotalLine

Ensure null values pass through so the engine returns unavailableMarket().

#43 - MLB Premium Experience Completion: Game Chips, Context Cards, and Market Wiring

What & Why
The MLB Live tab still does not meet the LiveLocks premium standard. The remaining issues are no longer simple styling bugs — they are product-completion issues across three areas:

1. Game chips are still not correctly displaying the actual matchup
2. Edge cards are still auto-populating with incorrect or incomplete data
3. Matchup details cards are too thin and do not surface enough decision context
4. MLB market coverage is still incomplete

This task upgrades MLB Live into a true premium product experience without touching protected engine-math files.

TASKS
--------------------------------------------------

1. Fix game chip matchup binding

In mlb-live.tsx and/or server/routes.ts:
- ensure the MLB games response includes:
  away abbreviation
  home abbreviation
  away full name
  home full name
  score
  inning/status
- frontend must render:
  AWAY @ HOME
  score inline
  inning/status below

If backend response shape is inconsistent, normalize it in server/routes.ts before sending to frontend.

Do NOT leave team extraction to ambiguous nested client parsing.

2. Enrich game chips with premium quick-glance context

For each game chip, add compact metadata rows/badges for:
- park factor (or park name if factor unavailable)
- probable / current starting pitcher
- weather summary (temp + wind shorthand, e.g. 74° / Out 9mph)

This must be compact, not oversized.
Match NBA density and premium feel.

If weather or pitcher is unavailable, show a neutral fallback badge rather than breaking layout.

3. Strictly scope and validate edge signals

Before rendering any signal card:
- selectedGameId must exist
- signal.gameId must match selectedGameId
- signal.playerId must exist
- signal.market must exist
- signal.line must not be null/undefined
- signal.enginePct must be a number
- signal.edge must be a number

Additionally:
- if the player in the signal does not exist in the selected game’s box score / lineup context, drop the card
- no fallback probability values anywhere
- if no valid signals remain, show clean empty state:
  “No live edges detected yet”

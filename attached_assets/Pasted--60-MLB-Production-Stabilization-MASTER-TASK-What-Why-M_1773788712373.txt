--------------------------------------------------
PHASE 1 — SIGNAL INTEGRITY (BLOCKER)
--------------------------------------------------

1. Remove all fallback probability logic
Search entire repo for:
- ?? 18
- || 18
- any hardcoded default probability constant
Delete every instance. The system must never fabricate a probability.

2. Enforce strict engine input validation
Before calling the engine for any player, require:
- playerId
- gameId
- battingOrderSlot (infer if needed)
- currentStats (AB, H, etc.)
- inning + top/bottom
If any are missing:
- SKIP PLAYER
- DO NOT CREATE SIGNAL

3. Build deterministic, isolated playerContext per player
Construct a fresh playerContext object for each player:
- playerId
- gameId
- battingOrderSlot
- currentStats
- inning
- isTopInning
- pitcherContext
- parkFactor
No shared references. No mutation across players.

4. Force independent engine execution
Call runEngine(playerContext) independently for each player.
If output is non-finite:
- SKIP PLAYER

5. Decouple probability from odds
Always compute probability first.
Then:
- if odds exist, compute edge
- if odds do not exist, edge = null
Return:
{
  playerId,
  gameId,
  probability,
  edge: nullable,
  odds: nullable
}
The pattern `if (!odds) return fallbackProbability` must not exist anywhere.

6. Fix signal array mapping integrity
Each signal must be a freshly constructed object.
No shared object reuse.
No array index mismatch bugs.

7. Add pre-response signal filtering in /api/mlb/live-signals
Drop any signal where:
- playerId missing
- probability non-finite
- probability < 0 or > 100
- gameId does not match requested game

8. Add temporary per-player debug logging
Log:
- playerId
- playerContext
- probability
for verification of uniqueness.

PHASE 1 SUCCESS CRITERIA

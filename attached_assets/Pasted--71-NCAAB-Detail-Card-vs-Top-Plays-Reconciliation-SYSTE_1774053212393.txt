#71 - NCAAB Detail Card vs Top Plays Reconciliation

## SYSTEM
NCAAB Service Layer / Engine Output / UI Binding

## OBJECTIVE
Fix the mismatch where NCAAB Top Plays and live game tiles show valid plays, but the selected game detail card renders Full Game / 1st Half / 2nd Half as "Unavailable" / "No data".

This is a reconciliation task between:
- the list/top-plays/live-games data source
- the selected game detail card data source

## CONTEXT
Observed runtime behavior:
- NCAAB Top Plays feed is populated
- Live game tiles are populated
- Opening certain live games (e.g. UCF @ UCLA, UNI @ SJU) shows:
  - Full Game unavailable
  - 1st Half unavailable
  - missing book implied %
  - no edge
- Therefore the engine is producing usable market data somewhere, but the detail card is not rendering the same usable market object

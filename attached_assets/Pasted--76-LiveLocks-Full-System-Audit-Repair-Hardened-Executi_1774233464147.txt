#76 - LiveLocks Full System Audit + Repair (Hardened Execution Version)

## What & Why
Perform a full stability and integrity audit across NBA, NCAAB, and MLB so the Engine → API → UI pipeline is verifiably healthy, gracefully degraded during odds failures, and MLB is production-ready before Wednesday. This task focuses on auditability, odds resilience, live-status normalization, safe engine execution, UI failsafes, analytics verification, and strict source-boundary enforcement.

This is a hardening task, not a model rewrite.

---

## Done looks like
- Structured temporary audit logs appear in server output for NBA, NCAAB, and MLB showing:
  - raw ingestion
  - processed lines
  - engineInput
  - engineOutput
- Odds API quota exhaustion, transient network failure, or empty-response conditions never blank the play feed when a last-known-good value exists
- Any stale-odds fallback always returns `isDegraded: true` in a consistent response shape
- MLB games in `"Live"`, `"In Progress"`, and `"in_progress"` all pass canonical live gating
- Engine never crashes or silently drops a market due to missing required fields; it logs skip reason and continues
- MLB prop key detection no longer depends on exact string equality
- UI renders:
  - `"Using last known lines"` when degraded data is shown
  - `"No strong edges right now"` when no plays qualify
  - never a blank or broken state
- ESPN is used only for score/state metadata; no odds or prop values flow from ESPN into engine inputs
- `/api/analytics/summary` returns valid responses for `today`, `7d`, and `30d`
- NBA play generation remains functionally unchanged except for audit visibility and shared guard-rail protections
- Temporary audit logs are either clearly gated behind a non-production/debug flag or removed at the end of the task

---

- New probability models
- Calibration rewrites

- Rewriting intentional NBA first-valid-line early exit if it is confirmed to be by design
- NCAAB H1/H2 loop work already covered by Task #67
- MLB premium UI / monetization changes

## Critical implementation rules
- Do backend/service audit and fixes first
- Do not patch UI to “look fixed” before route/service outputs are verified
- Do not introduce any new odds source
- Do not weaken existing NBA market extraction behavior unless required for shared safety logic
- Do not allow fallback codepaths to return empty arrays if cached data exists
- Any temporary logs must be removable or gated
- All degraded responses must use a consistent `isDegraded` contract

Add temporary structured logs at four checkpoints for all three sports:

1. Raw ingestion
   - bookmaker keys

   - game status
2. Post-processing
   - extracted lines
   - merged state
3. Pre-engine
   - final engineInput
4. Engine output

Format:
`[SPORT][GAME_ID] raw -> processed -> engineInput -> engineOutput`

Do not proceed to UI changes until these logs confirm actual service-layer behavior.

---

### Phase B — Odds resilience hardening
Audit and fix `server/oddsService.ts` so the last-known-good fallback is used on:
- quota exhaustion
- fetch/network errors
- malformed payloads
- empty responses where cache exists

Rules:
- Always return cached values when available
- Always mark returned stale data with `isDegraded: true`
- Never bypass cache on recoverable odds failures
- Never return empty odds arrays if cache exists for that game/market

Also verify whether fallback is:
- raw-odds level
- normalized-odds level
- both

If both exist, ensure they cannot drift or conflict.

---

### Phase C — MLB market and live-status hardening
Implement flexible matcher helpers in:
- `server/mlb/markets.ts`
- related MLB extraction files

Add:
- `isMLBPropKey()`
- any missing helper matchers needed for aliases / bookmaker variance

Replace exact string equality checks for MLB prop markets with matcher-based detection.

Then fix live status normalization in:
- `server/mlb/liveGameOrchestrator.ts`
- any route using raw `abstractState`

Canonical mapping must normalize:
- `"Live"` → `live`
- `"In Progress"` → `live`
- `"in_progress"` → `live`
- `"Pre-Game"` → `pregame`
- `"Final"` → `final`

Live gating must allow canonical `live`.

---

### Phase D — Engine input guard-rails
Before any engine run, validate required inputs for each sport:

Required:
- line
- odds
- minimum player/game context needed by that sport

If missing:
- log exact skip reason
- skip safely
- continue processing remaining markets/players
- do not throw
- do not silently fail

This must be implemented consistently across NBA, NCAAB, and MLB entry points.

---

### Phase E — ESPN source-boundary enforcement
Audit all routes/services that touch:
- props
- odds
- line values
- market prices

Ensure ESPN contributes only:
- scores
- game state / status
- schedule metadata if already intended

If any prop/odds/line value comes from ESPN:
- remove that dependency
- replace with odds-service output
- if unavailable, use cached degraded data or explicit disabled-market behavior

This is a source-of-truth cleanup step, not a UI step.

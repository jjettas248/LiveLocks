# MLB Engine Addendum — Poisson-Gamma Outcome Model

Objective
Enhance the MLB hits probability model by replacing the pure binomial outcome calculation with a Poisson-Gamma mixture model (Negative Binomial).

This captures real-world variance in hit outcomes that the binomial model underestimates.

The existing PA distribution model from Task #27 remains unchanged and feeds this model.

The architecture remains:

Game State → Opportunity Projection → Adjusted Hit Probability → Outcome Distribution → Expected Outcome → Edge vs Sportsbook

The only change is how the Outcome Distribution step is calculated.

STEP 1 — CREATE OUTCOME DISTRIBUTION MODULE

Create file:

server/mlb/outcomeDistribution.ts

Export function:

computeHitOutcomeProbability(
  remainingPA: number,
  adjustedHitRate: number,
  neededHits: number
)

Return:

number (probability of hitting OVER line)

--------------------------------------------------

STEP 2 — MODEL STRUCTURE

Instead of assuming hits follow a simple binomial:

Hits ~ Binomial(remainingPA, adjustedHitRate)

Use a Poisson-Gamma mixture:

λ ~ Gamma(shape = k, scale = θ)

Hits ~ Poisson(λ)

Resulting marginal distribution:

Hits ~ NegativeBinomial

This allows variance > mean.

--------------------------------------------------

STEP 3 — PARAMETERIZATION

Mean hits:

meanHits = remainingPA * adjustedHitRate

Variance inflation factor:

varianceMultiplier = 1.35

Variance:

variance = meanHits * varianceMultiplier

Compute dispersion parameter:

k = meanHits² / (variance - meanHits)

If k < 1, clamp to 1.

Compute success probability:

p = k / (k + meanHits)

--------------------------------------------------

STEP 4 — NEGATIVE BINOMIAL PROBABILITY

Compute probability of ≥ neededHits.

Use cumulative Negative Binomial distribution:

P(X ≥ neededHits)

Implementation approach:

Loop from neededHits to reasonable cap (e.g. 10 hits):

prob += negativeBinomialPMF(x, k, p)

--------------------------------------------------

STEP 5 — NEGATIVE BINOMIAL PMF

Implement helper:

negativeBinomialPMF(x, k, p)

Formula:

PMF(x) =
C(x + k - 1, x)
*
(1 - p)^x
*
p^k

Where C is combination.

Implement using log-gamma for numerical stability if available, otherwise factorial approximation.

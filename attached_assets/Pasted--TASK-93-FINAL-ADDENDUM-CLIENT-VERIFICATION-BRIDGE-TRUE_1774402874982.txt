## ROOT CAUSE:
The current implementation fails validation for two reasons:

1. The server-side success gate hardcodes:
- clientReceived = false
- quickViewRendered = false

with no real mechanism to ever flip them to true. Therefore the pipeline can never emit:
STATUS: HALFTIME PIPELINE RESTORED — TASK #50 UNBLOCKED

2. Quick View decoupling was not implemented in the actual render path. A cosmetic/legend change is insufficient. The validator needs proof that box-score rendering proceeds independently of halftime/2H availability and that any actual early-return or gating tied to those states has been removed.

## IMPLEMENTATION:

### Step 1: Add a real client verification bridge

Create a lightweight verification endpoint in the active NBA route layer, e.g. in server/routes.ts:

```ts
app.post("/api/halftime-plays/verify-client", (req, res) => {
  const { clientReceived, quickViewRendered, sourceCount, renderedCount } = req.body ?? {};

  globalThis.__halftimePipelineVerification = {
    clientReceived: clientReceived === true,
    quickViewRendered: quickViewRendered === true,
    sourceCount: Number(sourceCount) || 0,
    renderedCount: Number(renderedCount) || 0,
    verifiedAt: Date.now(),
  };

  console.log("[HT_CLIENT_VERIFICATION]", globalThis.__halftimePipelineVerification);

  res.json({ ok: true });
});

If a local module-scoped variable is better than globalThis in this codebase, use that. Keep it simple and active-path only.

Do not add auth complexity unless the route pattern already requires it.

Step 2: Use the verification bridge from the actual client flow

In the active halftime UI path in dashboard.tsx, immediately after the halftime fetch resolves and immediately after Quick View render input is finalized, post verification back to the server.

After fetch resolves, add:

console.log("[HT_CLIENT_ASSERT]", {
  received: data?.length ?? 0,
});

Then post:

await fetch("/api/halftime-plays/verify-client", {
  method: "POST",
  headers: { "Content-Type": "application/json" },
  body: JSON.stringify({
    clientReceived: (data?.length ?? 0) > 0,
    quickViewRendered: false,
    sourceCount: data?.length ?? 0,
    renderedCount: 0,
  }),
});

Then, in the actual Quick View render/useMemo/finalized dataset path, after rendered rows are computed, add:

console.log("[QUICK_VIEW_RENDER_ASSERT]", {
  renderedCount: renderedPlays?.length ?? 0,
});

and post:

await fetch("/api/halftime-plays/verify-client", {
  method: "POST",
  headers: { "Content-Type": "application/json" },
  body: JSON.stringify({
    clientReceived: true,
    quickViewRendered: (renderedPlays?.length ?? 0) > 0,
    sourceCount: sourcePlays?.length ?? 0,
    renderedCount: renderedPlays?.length ?? 0,
  }),
});

If using async directly in render is inappropriate, move this into the correct useEffect tied to the finalized rendered dataset. Keep it in the real active client path.

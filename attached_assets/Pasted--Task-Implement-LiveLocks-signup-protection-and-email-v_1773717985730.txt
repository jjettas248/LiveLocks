# Task: Implement LiveLocks signup protection and email verification system

We need to fix two problems:

1. Emails are not currently sending reliably.
2. Users are creating fake accounts to bypass the free-play limit.

Implement a full signup protection system that includes:

- email verification before plays
- reliable Resend email delivery
- disposable email blocking
- signup rate limiting
- optional captcha support

IMPORTANT:
Do NOT modify any sports engine logic, Stripe billing logic, or play calculation logic. Only modify signup/auth and email behavior.

--------------------------------------------------

FEATURE 1 — EMAIL VERIFICATION REQUIRED

Add a new column to the users table:

emailVerified: boolean (default false)

Also add:

emailVerificationToken: string | null

When a user registers:

1. create the user
2. generate a verification token
3. store token on the user
4. send verification email

Example token:

crypto.randomUUID()

Verification link:

/api/auth/verify-email?token=<token>

Create a route:

GET /api/auth/verify-email

Flow:

1. read token
2. find user with matching token
3. set emailVerified = true
4. clear emailVerificationToken
5. redirect to login page

--------------------------------------------------

FEATURE 2 — BLOCK UNVERIFIED USERS FROM USING PLAYS

In requirePlayAccess middleware:

if (!user.emailVerified) {
  return res.status(403).json({
    error: "Please verify your email to use LiveLocks."
  })
}

FEATURE 3 — SEND VERIFICATION EMAIL

Update server/email.ts

Add a function:

sendVerificationEmail(email, token)

Email content should include a verification link:

https://livelocksai.app/api/auth/verify-email?token=<token>

Use Resend:

resend.emails.send({
  from: "LiveLocks <team@livelocksai.app>",
  to: email,
  subject: "Verify your LiveLocks account",
  html: verification template
})

Add logging:

console.log("Verification email sent:", email)

Add error logging:

console.error("Verification email failed:", err)

FEATURE 4 — FIX EMAIL SENDING RELIABILITY

All email sends should use:

await sendEmailFunction()

Do NOT fire-and-forget.

Wrap with try/catch so signup does not fail if email fails.

Example:

try {
  await sendVerificationEmail(email, token)
} catch (err) {
  console.error("Email send error", err)
}

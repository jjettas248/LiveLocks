STEP 2 — FIX TAB ARRAY CONSTRUCTION

Find where the tabs array is built.
It currently likely looks something like:

  const tabs = [
    { id: "nba", label: "NBA Live", locked: false },
    { id: "ncaab", label: "NCAAB Live", 
      locked: !hasProAccess },
    { id: "analytics", label: "Analytics", 
      locked: false },
    { id: "mlb", label: "MLB Live", locked: true }
  ]

Three possible bugs — check all three:

BUG A: hasProAccess is called before user loads
  If user is null/undefined on first render,
  hasProAccess(undefined) returns false → locked.
  If session then loads asynchronously, the tab 
  may not re-render.
  
  Fix: use loading state
    const isLoading = !user && sessionStatus !== "error"
    
    if (isLoading) {
      // Render skeleton tabs or wait
      // Do NOT lock tabs during loading
    }

BUG B: hasProAccess receives wrong field
  User object may have tier nested differently:
    user.tier vs user.user?.tier vs 
    user.metadata?.tier vs user.publicMetadata?.tier
  
  Fix: try all locations:
    const userTier = 
      user?.tier ??
      user?.user?.tier ??
      user?.metadata?.tier ??
      user?.publicMetadata?.tier ??
      user?.subscription?.tier ??
      null
    
    const proAccess = hasProAccess(userTier)

BUG C: NCAAB tab conditionally hidden vs locked
  Tab may be HIDDEN (not rendered) rather than 
  locked (rendered with lock icon).
  
  Find if there is a condition like:
    {hasProAccess && <Tab id="ncaab" />}
  
  This removes the tab entirely for free users
  but also means a sync bug hides it for pro users.
  
  Fix: always render all tabs, control via locked prop:
    <Tab id="ncaab" locked={!proAccess} />
  
  Never conditionally remove NCAAB tab from DOM.
  Always render it. Only toggle locked state.

STEP 4 — ADD /api/me ENDPOINT AS FALLBACK

If session refresh is unreliable, add a direct 
user data endpoint that always returns fresh DB data:

  // /api/me  GET
  export async function GET(req) {
    // Get user identity from session/token
    const session = await getServerSession(req)
    // or however auth works in this codebase
    
    if (!session?.user?.email) {
      return Response.json(
        { error: "Not authenticated" }, 
        { status: 401 }
      )
    }
    
    // Always read from DB — never from session cache
    const user = await db.user.findUnique({
      where: { email: session.user.email },
      select: {
        id: true,
        email: true,
        tier: true,
        isNewProUser: true,
        upgradedAt: true,
        createdAt: true
      }
    })
    
    if (!user) {
      return Response.json(
        { error: "User not found" }, 
        { status: 404 }
      )
    }
    
    return Response.json({
      ...user,
      normalizedTier: normalizeTier(user.tier),
      hasProAccess: hasProAccess(user.tier)
    })
  }

  Call /api/me on mount to get fresh tier:

ADDENDUM — Probability Pipeline Hardening

Purpose
Guarantee sportsbook implied probabilities can never reach a UI display component, even if future code changes occur. The engine probability must remain the single source of truth across the application.


PART 1 — Normalize API return structure

File: server/storage.ts

Inside calculateProbability(), confirm every return path includes:

return {
  probability,
  impliedProbability: probability,
  edge: probability - marketImpliedProb,
  ...
}

Rules:
• marketImpliedProb is used ONLY for edge calculation.
• marketImpliedProb must never be returned as a field.
• impliedProbability exists only for backward compatibility and must always equal probability.

Before the return block ensure probability is normalized:

probability = Math.max(2, Math.min(98, probability))


PART 2 — Remove impliedProbability from client-facing types

File: shared/schema.ts

Remove impliedProbability from any interface or type used by frontend UI components.

Example UI model:

export interface PlayResult {
  probability: number
  edge: number
  direction: "OVER" | "UNDER"
  projection?: number
  liveLine?: number
}

The backend may still compute impliedProbability internally, but the TypeScript types exposed to the UI must not include it.

This allows the TypeScript compiler to block accidental UI usage.


PART 3 — Restrict ProbabilityRing component props

File: client/src/components/probability-ring.tsx

Update props type to:

type ProbabilityRingProps = {
  probability: number
}

Remove any props named:

impliedProbability
marketProbability
bookProbability
sportsbookProbability

All call sites must pass:

<ProbabilityRing probability={play.probability} />


PART 4 — Dashboard runtime guard

File: client/src/pages/dashboard.tsx

Inside the play card render loop add a dev-only guard:

if (process.env.NODE_ENV !== "production" && play.probability == null) {
  console.error("Missing engine probability for play", play)
}

This prevents silent rendering failures if a play object is malformed.


PART 5 — Probability display verification

Confirm the following components render probability using ONLY the engine field:

ProbabilityRing → play.probability
Edge cards → play.probability
Halftime play cards → play.probability
Quick signal badges → play.probability
Matchup calculator → result.probability
MLB live cards → play.probability
Parlay slip → leg.probability

No UI component may read:

marketImpliedProbability
bookProbability
sportsbookProbability
impliedProbability

except inside internal edge math logic.


PART 6 — Remove temporary debug logs

Ensure no debug logs remain in production code.

Remove if present:

console.log("Engine probability displayed:", play.probability)


VALIDATION

Implementation is complete when:

• TypeScript throws an error if UI references impliedProbability
• Every probability display reads play.probability or result.probability
• Edge math still uses marketImpliedProb internally
• No sportsbook implied probability reaches any UI component
• Application compiles successfully

ADDENDUM — Resend Email Wiring Verification

The project already includes a Resend integration, but the Resend dashboard shows "No sent emails yet", which indicates that email functions are not executing or not reaching the Resend API.

We need to explicitly verify and wire the Resend client so all lifecycle and verification emails are actually sent.

IMPORTANT:
Do NOT modify sports engines, alert systems, Stripe billing, or probability calculations.

Only update the email delivery wiring.

--------------------------------------------------

STEP 1 — VERIFY RESEND CLIENT INITIALIZATION

Inspect:

server/email.ts

Confirm the Resend client is initialized exactly once:

import { Resend } from "resend"

const resend = new Resend(process.env.RESEND_API_KEY)

If the key is missing:

console.warn("RESEND_API_KEY missing")

but do not crash the server.

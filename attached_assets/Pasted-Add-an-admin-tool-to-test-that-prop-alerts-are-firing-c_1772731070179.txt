ON SEND — TEST ALERT FLOW

When "Send Test Alert" is clicked:

  Build notification payload exactly as production does:
  
  const testPlay = {
    playerName: formData.playerName,
    team: formData.team,
    market: formData.market,
    direction: formData.direction,
    line: parseFloat(formData.line),
    engineProb: parseFloat(formData.confidence),
    gameId: "test-" + Date.now(),
    playerId: "test-player",
    sport: "nba"
  }
  
  const title = getNotificationTitle(testPlay.engineProb)
  if (!title) {
    showToast("Confidence below 80% threshold", "red")
    return
  }
  
  const directionLabel = 
    testPlay.direction === "Over" ? "Over" : "Under"
  
  const body = 
    `${testPlay.playerName} — ` +
    `${directionLabel} ${testPlay.line} ` +
    `${testPlay.market} — ` +
    `${testPlay.engineProb}% confidence`
  
  POST to /api/admin/test-alert:
    {
      title,
      body,
      target: formData.sendTo,
      // "self" or "all"
      testPlay
    }

SERVER: /api/admin/test-alert

  If target === "self":
    Send Web Push notification only to the 
    admin user's own push subscription
    
    Find admin's push subscription in DB:
      WHERE userId = currentAdminId
      AND pushSubscription IS NOT NULL
    
    Send using existing web push sender function
    with the test payload
    
    Do NOT record this as a real play in DB
    Add flag: isTest: true in notification data

  If target === "all":
    Require confirmation — return 
    { requiresConfirmation: true } first
    
    Admin sees modal:
      "Send test alert to all [N] subscribers?"
      [Cancel]  [Confirm Send]
    
    On confirm: send to all push subscriptions
    Still mark as isTest: true
    Still do NOT record in plays DB

RESPONSE HANDLING:

  Success:
    showToast("✓ Test alert sent", "teal")
    Show in right panel below preview:
      "✓ Sent [time]"
      "Title: [title]"
      "Body: [body]"
      "Delivered to: [self / N subscribers]"
    
  Error — no push subscription for admin:
    showToast(
      "No push subscription found. " +
      "Install app to home screen first.", 
      "amber"
    )
  
  Error — below 80% threshold:
    showToast(
      "Confidence below 80% — " +
      "alert would not fire in production", 
      "red"
    )

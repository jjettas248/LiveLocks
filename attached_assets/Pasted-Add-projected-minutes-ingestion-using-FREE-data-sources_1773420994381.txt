Add projected minutes ingestion using FREE data sources first, with automatic fallback to RotoWire when ROTOWIRE_API_KEY is added.

Goal:
Provide projectedMinutes for players daily so the LiveLocks engine models rotations more accurately without requiring a paid API immediately.

Do NOT modify:
calculateProbability()
minutesModel.ts core logic
API routes

Only add projection ingestion services.


STEP 1 — Extend Player Schema

In shared/schema.ts ensure players table includes:

projectedMinutes?: number
projectionSource?: string
projectionUpdatedAt?: Date


STEP 2 — Create Projection Service

Create file:

server/services/minutesProjectionService.ts

This service will attempt projections in priority order:

1. Sleeper API (free)
2. RotoWire projections page scrape (free)
3. Official RotoWire API (when key is available)

The service should return:

{
  playerName: string,
  projectedMinutes: number,
  source: string
}


STEP 3 — Sleeper Projection Pull (Primary Free Source)

Fetch:

https://api.sleeper.app/projections/nba

Parse fields:

player_name
minutes

Map to:

projectedMinutes


STEP 4 — RotoWire Web Projection Scrape (Secondary Free Source)

Fetch:

https://www.rotowire.com/basketball/projections.php

Parse table rows to extract:

player name
team
minutes

Normalize names to match DB player names.

Return projectionSource = "rotowire_scrape"


STEP 5 — Official RotoWire API (Activated When Key Exists)

If environment variable ROTOWIRE_API_KEY exists:

fetch:

https://api.rotowire.com/nba/projections.php?key=${ROTOWIRE_API_KEY}&format=json

Fields:

player_name
proj_minutes

Set projectionSource = "rotowire_api"

This overrides the free projections.


STEP 6 — Projection Priority Logic

Use projections in this order:

if rotowire_api exists → use it
else if sleeper projection exists → use it
else if rotowire scrape exists → use it
else fallback to avgMinutes


STEP 7 — Update Players Table

Match players by name.

Update:

players.projectedMinutes
players.projectionSource
players.projectionUpdatedAt


STEP 8 — Sync Schedule

Run projection sync:

• on server startup
• every 30 minutes before games
• via admin endpoint

Create admin endpoint:

POST /api/admin/sync-minutes-projections


STEP 9 — Engine Integration

minutesModel.ts must use:

baseRotationMinutes =
  projectedMinutes ?? avgMinutes


STEP 10 — Debug Output

Add debug fields inside calculateProbability():

projectedMinutes
projectionSource

Values:

"sleeper"
"rotowire_scrape"
"rotowire_api"
"season_avg"


STEP 11 — Validation

Confirm:

Projected minutes populate for players before games start.

Bench players with expanded roles reflect projectedMinutes instead of avgMinutes.

When ROTOWIRE_API_KEY is added, the system automatically switches to the official RotoWire feed.

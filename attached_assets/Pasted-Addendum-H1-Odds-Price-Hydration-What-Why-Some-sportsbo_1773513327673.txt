Addendum — H1 Odds Price Hydration

What & Why
Some sportsbook feeds provide first-half markets with lines but omit odds extraction because the ingestion logic assumes a specific outcome structure. When odds are missing, the engine cannot compute implied probability, resulting in displays such as "Engine 78% vs Book —%".

The ingestion layer must reliably extract sportsbook prices for first-half totals and spreads whenever they exist.

Done looks like
When sportsbooks provide H1 totals or spreads with prices, the engine receives non-null odds values
Book implied probability populates correctly in H1 market rows
The UI displays both line and book probability instead of "—%"
If sportsbooks genuinely do not provide prices, the engine continues gracefully without errors

Out of scope
Synthetic line generation
Changes to CLV logic
Changes to public fade logic
Changes to projections
UI redesign

Tasks

Add tolerant outcome parsing

When extracting markets from SGO or The Odds API, do not assume fixed outcome ordering. Instead, search outcomes dynamically.

Example:

const overOutcome = market.outcomes.find(o =>
  normalizeMarketKey(o.name).includes("over")
)

const underOutcome = market.outcomes.find(o =>
  normalizeMarketKey(o.name).includes("under")
)

const homeOutcome = market.outcomes.find(o =>
  normalizeMarketKey(o.name).includes("home")
)

const awayOutcome = market.outcomes.find(o =>
  normalizeMarketKey(o.name).includes("away")
)

Extract odds when present

If outcome.price exists, assign:

h1TotalOverOdds = overOutcome?.price ?? null
h1TotalUnderOdds = underOutcome?.price ?? null

h1SpreadHomeOdds = homeOutcome?.price ?? null
h1SpreadAwayOdds = awayOutcome?.price ?? null

Ensure prices propagate to engine input

Confirm that runNCAABEngine receives:

h1TotalOverOdds
h1TotalUnderOdds
h1SpreadHomeOdds
h1SpreadAwayOdds

These values are used by the engine to compute implied probability.

Add diagnostics

If h1TotalLine exists but both odds values are null:

console.warn("H1 odds hydration failure", {
  gameId,
  marketKey,
  outcomes
})

Non-production logging

If NODE_ENV !== "production":

console.debug("H1 market outcomes detected", {
  gameId,
  outcomes
})

Safety behavior

If odds cannot be extracted but lines exist, the engine should still display the line but leave book probability null.

Addendum — Robust Outcome & Odds Extraction

What & Why
Even when first-half markets are detected and lines are extracted, odds prices can still fail to hydrate because sportsbooks do not always label outcomes consistently.

Common variations include:

• using team names instead of "home"/"away"
• returning odds as "price" or "odds"
• inconsistent ordering of outcomes

This addendum ensures outcome detection and odds extraction are resilient across sportsbook feed formats.

Done looks like
Spread outcomes are correctly identified even when sportsbook feeds use team names instead of "home" or "away"
Odds prices populate correctly whether feeds use "price" or "odds"
No H1 spread or total markets fail because of outcome naming differences
Engine receives valid H1 odds values whenever sportsbooks provide them

Out of scope
Synthetic line generation
Changes to CLV logic
Changes to public fade logic
Changes to projections
UI changes

Tasks

Flexible spread outcome detection

When parsing H1 spread markets, attempt detection in this order:

1. outcomes containing "home"
2. outcomes containing "away"
3. outcomes containing the normalized home team name
4. outcomes containing the normalized away team name

Example logic:

const homeOutcome =
  outcomes.find(o =>
    normalizeMarketKey(o.name).includes("home")
  ) ||
  outcomes.find(o =>
    normalizeMarketKey(o.name).includes(normalizeMarketKey(homeTeam))
  )

const awayOutcome =
  outcomes.find(o =>
    normalizeMarketKey(o.name).includes("away")
  ) ||
  outcomes.find(o =>
    normalizeMarketKey(o.name).includes(normalizeMarketKey(awayTeam))
  )

Flexible odds extraction

Sportsbook feeds may return odds in either:

price
or
odds

Create helper:

function extractPrice(outcome) {
  return outcome?.price ?? outcome?.odds ?? null
}

Use this helper when assigning H1 odds values:

h1TotalOverOdds = extractPrice(overOutcome)
h1TotalUnderOdds = extractPrice(underOutcome)

h1SpreadHomeOdds = extractPrice(homeOutcome)
h1SpreadAwayOdds = extractPrice(awayOutcome)

Outcome classification priority

To prevent collisions between market classifiers:

Always evaluate total markets first, then spreads.

Example:

if (isH1TotalKey(key)) {
  // process total market
} else if (isH1SpreadKey(key)) {
  // process spread market
}

This ensures keys containing both "total" and "spread" tokens do not produce duplicate classifications.

Diagnostics

If H1 spread markets are detected but no home/away outcomes can be identified, emit:

console.warn("H1 spread outcome detection failure", {
  gameId,
  marketKey,
  outcomes
})

In non-production environments log detected outcomes:

if (process.env.NODE_ENV !== "production") {
  console.debug("H1 spread outcomes detected", {
    gameId,
    outcomes
  })
}

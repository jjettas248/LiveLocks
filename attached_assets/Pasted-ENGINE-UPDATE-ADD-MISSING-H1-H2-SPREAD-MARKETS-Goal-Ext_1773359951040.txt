Important constraints
- Do NOT modify any UI files
- Do NOT change existing probability models
- Do NOT modify total or team total logic
- Only extend the engine verdict generation
- Follow the existing marketVerdicts structure
- Respect existing guardrails: do not fabricate markets when book lines are missing

Files to modify
server/ncaabEngine.ts
(server/ncaabService.ts only if necessary to pass book lines through)

STEP 1 — Locate spread verdict logic

In server/ncaabEngine.ts find the section that pushes spread verdicts into:

STEP 2 — Add H1 spread projection

Before generating the verdict, compute the projected H1 spread.

Use existing H1 team projections already produced by the engine:

proj1HTeamTotalHome
proj1HTeamTotalAway

Compute:

proj1HSpread = proj1HTeamTotalHome - proj1HTeamTotalAway

Use the same probability distribution model already used for the full-game spread.

STEP 5 — Push H1 spread verdict

If both projection and book line exist:

push this object into engineOutput.marketVerdicts:

  market: "h1_spread",
  line: book1HSpreadLine,
  overProb: prob1HSpreadCover,
  underProb: 100 - prob1HSpreadCover,
  projection: proj1HSpread
}

STEP 6 — Add H2 spread projection

Compute projected second-half margin.

Inputs already available:

currentScoreMargin

currentScoreMargin =
homeScore - awayScore

Compute:

proj2HSpread =
finalProjectedSpread - currentScoreMargin

This represents the expected spread margin for the remaining half.

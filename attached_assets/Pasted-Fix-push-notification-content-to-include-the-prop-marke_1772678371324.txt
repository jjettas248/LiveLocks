FIX NOTIFICATION PAYLOAD SHAPE

Every notification payload must include prop market.

Current payload likely looks like:
  {
    title: "LiveLocks High-Confidence Play",
    body: `${playerName} ${direction}${line} — 
           ${confidence}% confidence. Tap to view.`
  }

Updated payload:

  {
    title: "🔒 LiveLocks · High Confidence",
    body: `${playerName} — ${directionLabel} ${line} 
           ${marketLabel} — ${confidence}% confidence`,
    icon: "/icon-192x192.png",
    badge: "/badge-72x72.png",
    tag: `livelocks-${playId}`,
    data: {
      url: "/",
      playId: playId,
      sport: sport
    }
  }

WHERE:

  directionLabel:
    "O" or "Over" in raw data → "Over"
    "U" or "Under" in raw data → "Under"
    
    const directionLabel = 
      direction?.toLowerCase().startsWith("o") 
        ? "Over" : "Under"

  marketLabel:
    Must come from the prop/market field on the play object
    Check what field name stores prop type in play data:
      common field names: prop, market, stat, propType, 
                          category, betType
    
    const marketLabel = 
      play.prop || 
      play.market || 
      play.stat || 
      play.propType || 
      play.category ||
      ""
    
    If marketLabel is empty string:
      Do not show it — omit from body
      Body becomes: 
        "${playerName} — ${directionLabel} ${line} 
         — ${confidence}% confidence"
    
    If marketLabel exists:
      Body:
        "${playerName} — ${directionLabel} ${line} 
         ${marketLabel} — ${confidence}% confidence"

  Examples of correct output:
    "Jared McCain — Under 11.5 Points — 92% confidence"
    "Chet Holmgren — Over 28 Rebounds+Assists — 98% confidence"
    "SGA — Under 31.5 Points — 92% confidence"
    "Jalen Brunson — Under 19.5 Points — 92% confidence"

FIX SERVICE WORKER NOTIFICATION DISPLAY

In the service worker file (sw.js or similar):

  self.addEventListener("push", event => {
    const data = event.data?.json() ?? {}
    
    const options = {
      body: data.body,
      icon: data.icon || "/icon-192x192.png",
      badge: data.badge || "/badge-72x72.png",
      tag: data.tag || "livelocks-notification",
      renotify: true,
      // renotify: true means each new notification 
      // shows even if same tag — important for 
      // multiple plays at same time
      data: data.data || { url: "/" },
      actions: [
        {
          action: "view",
          title: "View Play"
        },
        {
          action: "dismiss",
          title: "Dismiss"
        }
      ]
    }
    
    event.waitUntil(
      self.registration.showNotification(data.title, options)
    )
  })

  // Handle notification click
  self.addEventListener("notificationclick", event => {
    event.notification.close()
    
    if (event.action === "dismiss") return
    
    const url = event.notification.data?.url || "/"
    
    event.waitUntil(
      clients.matchAll({ type: "window" }).then(clientList => {
        // If app already open, focus it
        for (const client of clientList) {
          if (client.url === url && "focus" in client) {
            return client.focus()
          }
        }
        // Otherwise open new window
        if (clients.openWindow) {
          return clients.openWindow(url)
        }
      })
    )
  })

DEDUPLICATION

Current issue in screenshot: same player sent 
multiple times (Chet Holmgren appears 3 times).

Add deduplication before sending notifications:

  // Track sent notifications in memory
  const sentNotifications = new Set()
  
  const getNotificationKey = (play) => 
    `${play.playerId}-${play.market}-${play.line}-${play.gameId}`
  
  // Before sending:
  const key = getNotificationKey(play)
  if (sentNotifications.has(key)) return
  sentNotifications.add(key)
  
  // Clear set on new game day
  // (reset at midnight or on new scoreboard fetch)

This prevents the same play from being sent 
multiple times when the engine refreshes.

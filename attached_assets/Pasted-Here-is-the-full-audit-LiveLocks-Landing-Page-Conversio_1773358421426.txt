LiveLocks Landing Page — Conversion & Credibility Audit
Section 1 — Conversion Audit
Hero Clarity
The headline "Stop guessing on live props" is punchy and directional, but it does not name the mechanism. A bettor landing here for the first time cannot immediately tell whether this is a picks service, a calculator, or a dashboard tool. The sub-headline corrects that — but it is one long sentence doing three jobs at once (scans, ranks, texts), which dilutes each one. Readers skim; they will catch one of the three ideas or none.

The live status badge ("Detection engine is live — NBA season active") is the single most credibility-building element on the page. It signals the product is running right now. It should be bigger, not treated as a minor status chip.

Product Credibility
The dashboard preview is a coded component — it looks like a prototype. Bettors are skeptical audiences. A simulated UI feels like a pitch deck, not a real product. The SMS section uses an actual phone image with realistic message content, and it reads as far more trustworthy as a result. The contrast between the two undermines confidence in the dashboard itself.

Perceived Product Complexity
There are two separate "features" sections — the feature cards grid and the secondary "Top 2H plays" breakdown below it. Combined, they describe the same product twice using different framing. This creates the impression of a complex tool, when the core pitch is actually simple: halftime hits your phone with the best prop edges. Redundancy reads as padding.

Emotional Hook
The strongest emotional moment is the SMS section — speed, no friction, "you do not need to have the dashboard open." That is the real value proposition for a bettor who wants an edge without extra work. It is buried mid-page.

CTA Strength
"Sign Up Free" is fine, but "No credit card required · 15 free probability calculations included" immediately anchors the user to a limit. The free tier's value should be framed as access, not a countdown.

What reduces trust: Simulated dashboard UI instead of a real screenshot.
What causes friction: Two overlapping features sections. Long sub-headline in the hero.
What prevents signups: The best pitch (SMS, instant, no-friction) is not the first thing seen after the headline.

Section 2 — Dashboard Preview Analysis
SMS Section — Why It Feels More Compelling
The phone mockup works because it shows a real artifact (a text message) with specific detail (player name, prop, edge percentage, probability). It is immediately scannable. There is no ambiguity about what the product does — you read the message and you understand the output.

Dashboard Preview — Why It Underperforms
The dashboard component is dense, interactive-looking, and custom-built — which should be an advantage. But because it is rendered in code rather than shown as a real screenshot, it carries an uncanny quality. The tabs (NBA Live, 2H Plays, NCAAB), the prop cards, the hit probability percentages — all of that is real signal, but it reads as a demo. The moment a user senses "this was built for the landing page," trust erodes.

UI Density	High — tabs, sidebar, nav	Low — three SMS messages
Information Hierarchy	Unclear — sidebar competes with cards	Clear — one message per play

Recommendation: Hybrid (Option 3)
Replace the coded component with a real screenshot of the live dashboard cropped to show 2–3 prop edge cards only, no sidebar, no navigation chrome. Add a browser frame around it for context. The screenshot makes it real; the crop keeps it digestible. This is the same technique the SMS section uses — show the output, not the interface.

Section 3 — Visual Hierarchy Problems
Spacing
The page stacks sections at roughly equal visual weight. Nothing creates a breathing moment that says "pause here, this is the core idea." The hero → dashboard transition happens immediately, before the user has internalized the pitch.

Section Order
Current order:

CTA
The SMS Alerts section is the strongest single-concept pitch on the page — "best plays, texted to you at halftime" — but it appears fifth. It should appear second, directly after the hero, as immediate proof of the product's core value delivery.

Information Overload
The features grid (3 cards) followed immediately by the secondary features breakdown ("Probability edge scoring," "Transparency," "One-click parlay building") is repetitive. Both sections describe the same product from slightly different angles. Neither is doing distinct work.

What should be simplified or removed:

Collapse the two features sections into one. Three bullets maximum.
The "All Sports ($65/mo)" tier adds noise. If MLB live predictions are not fully built, including this tier introduces doubt about the core product's completeness.
Which section currently sells the product best:
The SMS Alerts section. It has a single clear output, a recognizable format, explicit benefit language, and a physical artifact. It is the most convincing proof that the product is real and useful.

Change 6:
Make the live status badge in the navbar larger and more prominent in the hero — either repeat it inline below the sub-headline or increase its visual weight. It is the single strongest trust signal on the page and it is being treated as decoration.

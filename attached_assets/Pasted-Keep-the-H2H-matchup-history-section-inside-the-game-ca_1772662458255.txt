Keep the H2H matchup history section inside the game card 
after it transitions from pre-game to live state.

Default state when live card opens: H2H section collapsed.
User must manually expand it.

TOGGLE STRUCTURE:

Placement: 
  Between the stat grid and the verdict rows in the live card.
  Same position it occupied in the pre-game card.

Collapsed state (default):
  Single full-width row:
    Left: "Matchup History" text-sm zinc-400
    Right: chevron-down icon zinc-500
    bg: zinc-900/50
    border: zinc-800
    rounded-lg
    cursor pointer
    hover: border zinc-600, text zinc-300

  No H2H rows visible.

Expanded state (user toggles open):
  Chevron rotates 180deg (300ms ease)
  H2H rows slide down (max-height transition, 300ms ease)
  All 3 matchup rows visible with spread + O/U badges
  "Matchup History" label stays visible as section header

Toggle state:
  const [h2hOpen, setH2hOpen] = useState(false)
  // false = collapsed by default in both pre-game and live card

PRE-GAME CARD BEHAVIOR:
  In the pre-game card the H2H section is EXPANDED by default
  (h2hOpen starts true for pre-game cards)
  
  Rationale: pre-game is research mode, user wants context.
  Live card is action mode, H2H is secondary reference.

  When card transitions pre-game → live:
    setH2hOpen(false)  // collapse it on transition
    
  This gives a clean transition — live card opens focused 
  on the gauge and verdict rows, history tucked away.

DATA PERSISTENCE:
  H2H data fetched once during pre-game state.
  Cache it on the game object so live card does not 
  re-fetch — data is already loaded, just hidden.
  
  const [h2hData, setH2hData] = useState(null)
  
  // Fetch once on pre-game card mount
  // Store result in h2hData
  // Pass h2hData to live card on transition
  // Live card reads from cache, never re-fetches

  On fetch error in pre-game:
    h2hData = [] (empty array)
    Toggle still renders but expanded state shows:
      "Matchup history unavailable"
    Do not retry in live card.

TOGGLE ANIMATION:
  Use max-height transition pattern:
    collapsed: max-height 0, overflow hidden, opacity 0
    expanded: max-height 400px, overflow visible, opacity 1
    transition: max-height 300ms ease, opacity 200ms ease
  
  Do not use layout shift or reflow — 
  keep transition smooth during live card updates.

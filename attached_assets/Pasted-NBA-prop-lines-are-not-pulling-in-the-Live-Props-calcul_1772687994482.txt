STEP 2 — FIX PLAYER ID RESOLUTION

Most props APIs require a numeric or string player ID,
not a display name like "Pascal Siakam (PF)".

FIND how player ID is currently resolved.

If player name is being sent directly to the API:
  This will always fail — fix it.

CORRECT APPROACH:
  Use ESPN player ID from the roster data already 
  loaded for the game.

  The game object from ESPN already contains 
  player roster via:
    `https://site.api.espn.com/apis/site/v2/sports/` +
    `basketball/nba/summary?event=${gameId}`
  
  From summary: response.rosters → array of players
  Each player has: athlete.id, athlete.displayName

  Player ID resolution:
  
    const resolvePlayerId = (playerName, gameId) => {
      // Clean display name: remove position "(PF)"
      const cleanName = playerName
        .replace(/\s*\([A-Z]+\)\s*$/, "")
        .trim()
      // e.g. "Pascal Siakam (PF)" → "Pascal Siakam"
      
      // Find in roster data already loaded
      const roster = gameRosters[gameId] ?? []
      const player = roster.find(p => 
        p.athlete?.displayName
          ?.toLowerCase()
          .includes(cleanName.toLowerCase()) ||
        cleanName.toLowerCase()
          .includes(
            p.athlete?.displayName?.toLowerCase()
              ?.split(" ")?.[1] ?? ""
          )
      )
      
      return player?.athlete?.id ?? null
    }

  If roster not yet loaded for this game:
    Fetch it first:
    
    const fetchGameRoster = async (gameId) => {
      const url = 
        `https://site.api.espn.com/apis/site/v2/` +
        `sports/basketball/nba/summary?event=${gameId}`
      const res = await fetch(url)
      const data = await res.json()
      
      const allPlayers = []
      data.rosters?.forEach(team => {
        team.roster?.forEach(player => {
          allPlayers.push(player)
        })
      })
      
      setGameRosters(prev => ({
        ...prev,
        [gameId]: allPlayers
      }))
      
      return allPlayers
    }

NCAAB UI CONTRACT GUARDRAILS — ENFORCEMENT RULES

Purpose
Prevent silent regressions where the UI substitutes full-game values, legacy fields, or engine projections in place of the correct market-specific verdicts.

These guardrails apply to all NCAAB UI rendering paths.

----------------------------------------------------

1. MARKET ISOLATION RULE

Each tab may only read verdicts for its own market scope.

Allowed verdict reads:

Full Game tab
- total
- spread
- team totals (full-game only)

H1 tab
- h1_total
- h1_spread

H2 tab
- h2_total
- h2_spread

If the requested verdict does not exist:

Render:
- "No data" for probabilities
- "No line" for spread/total

Do NOT:
- substitute full-game verdicts
- substitute hard-coded values (e.g. 50)
- substitute engine projections

Emit dev-only warning:

console.warn("NCAAB market verdict missing for active tab", {
  gameId: play.gameId,
  activeTab: marketTab,
  requestedMarket,
  availableVerdicts: play.engineOutput?.marketVerdicts
})

----------------------------------------------------

2. STRICT MARKET MATCHING

The helper resolveVerdictForMarket(engineOutput, market) must only return a verdict where:

mv.market === requestedMarket

Never match partial or similar keys.

Examples of INVALID matches:
spread → h2_spread
spread → h1_spread
h2_spread → spread

If no exact match exists, return null.

----------------------------------------------------

3. SINGLE SOURCE OF TRUTH

UI probability and market rendering must originate only from:

engineOutput.marketVerdicts
engineOutput.displayOutput

Legacy play-level fields must NOT be used for UI logic or display:

play.overProb
play.spreadProb
play.h2EngineOverProb
play.spreadEdge
play.totalEdge

These may remain in the play object for analytics but cannot drive UI rendering.

----------------------------------------------------

4. DISPLAY VS NUMERIC CONTRACT

Numeric logic must use calibrated numeric probabilities:

engineOutput.calibratedOverProb
engineOutput.calibratedUnderProb
engineOutput.calibratedSpreadProb

User-facing text must use engine formatted values when available:

engineOutput.displayOutput.displayProbability
engineOutput.displayOutput.overProb
engineOutput.displayOutput.underProb
engineOutput.displayOutput.spreadProb

Do NOT derive display strings from numeric values if a displayOutput value exists.

----------------------------------------------------

5. BOOK LINE VS ENGINE ESTIMATE

If a sportsbook line is unavailable:

Do NOT render projected totals as if they are sportsbook lines.

Instead render clearly labeled estimate:

"Engine est: XXX.X"

This rule applies to:

effectiveFGLine
effective1HLine
effective2HLine

----------------------------------------------------

6. SPREAD SIGN CONSISTENCY

Spread sign must follow the same logic used server-side.

Primary source:
play.spreadTeam

Fallback:
case-insensitive comparison of play.favorite vs team names.

Example:

favorite === homeTeam → homeSpread = -abs(spread)
favorite === awayTeam → awaySpread = -abs(spread)

Never:
- assume home favorite
- default spread sign when team cannot be resolved.

----------------------------------------------------

7. DEV SAFETY WARNINGS

When any guardrail condition is triggered, emit development-only warnings.

Examples:

console.warn("NCAAB market verdict missing", {...})

console.warn("NCAAB spread fallback triggered", {...})

Warnings must run only when:

NODE_ENV !== "production"

----------------------------------------------------

END GUARDRAILS

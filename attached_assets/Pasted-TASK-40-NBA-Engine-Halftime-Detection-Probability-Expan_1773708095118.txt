PART 1 — HALFTIME DETECTION FIX
========================================================

File
server/routes.ts

Locate the GET /api/halftime-plays route.

Find the current halftime detection logic that sets:

isHalftime


Replace the current boolean with the following logic.


First ensure a helper exists:

function parseClockToSeconds(clock: string | null | undefined): number {
  if (!clock) return 0;

  if (clock.includes(":")) {
    const [m, s] = clock.split(":").map(Number);
    return m * 60 + s;
  }

  if (clock.includes(".")) {
    return Math.floor(parseFloat(clock));
  }

  return Number(clock) || 0;
}



Then replace halftime detection with:

const clockSeconds = parseClockToSeconds(game.clock);

const isHalftime =
  (game.period === 2 && clockSeconds <= 10) ||
  game.status === "HALF" ||
  game.status === "HALFTIME" ||
  (game.period === 3 && clockSeconds === 720);



If isHalftime is true, include the game in halftimeGames.



Add debug logging:

console.log("HALFTIME DETECTED:", {
  home: game.homeTeam,
  away: game.awayTeam,
  period: game.period,
  clock: game.clock
});



Expected behavior

Games showing:

period = 2
clock = "0.0"

must now be treated as halftime.

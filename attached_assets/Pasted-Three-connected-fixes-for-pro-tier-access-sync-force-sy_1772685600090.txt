STEP 2 — FIX SESSION SYNC BASED ON AUTH SYSTEM FOUND

Use ONLY the auth system identified in Step 1.
Do not introduce a different auth library.

IF NextAuth:

  In [...nextauth] route, add tier to session:
    callbacks: {
      async session({ session, token }) {
        // Always re-read tier from DB on session check
        const user = await db.user.findUnique({
          where: { email: session.user.email },
          select: { 
            tier: true, 
            requiresRefresh: true,
            isNewProUser: true
          }
        })
        session.user.tier = 
          normalizeTier(user?.tier)
        session.user.isNewProUser = 
          user?.isNewProUser ?? false
        
        // Clear refresh flag after reading
        if (user?.requiresRefresh) {
          await db.user.update({
            where: { email: session.user.email },
            data: { requiresRefresh: false }
          })
        }
        return session
      }
    }

  Client re-fetch on focus:
    const { data: session, update } = useSession()
    useEffect(() => {
      const onFocus = () => update()
      window.addEventListener("focus", onFocus)
      return () => 
        window.removeEventListener("focus", onFocus)
    }, [update])

IF Clerk:

  Use Clerk's publicMetadata to store tier:
    In /api/admin/change-tier after DB write:
      await clerkClient.users.updateUserMetadata(
        user.clerkId, {
          publicMetadata: { 
            tier: newTierKey,
            isNewProUser: true
          }
        }
      )
  
  Client reads from:
    const { user } = useUser()
    const tier = normalizeTier(
      user?.publicMetadata?.tier
    )
    const isNewProUser = 
      user?.publicMetadata?.isNewProUser ?? false

IF Supabase Auth:

  Store tier in user_metadata or a users table:
    After admin change:
      await supabase.auth.admin.updateUserById(
        userId, {
          user_metadata: { 
            tier: newTierKey,
            isNewProUser: true
          }
        }
      )
  
  Client reads:
    const { data: { user } } = 
      await supabase.auth.getUser()
    const tier = normalizeTier(
      user?.user_metadata?.tier
    )

IF Custom JWT:

  Add tier to JWT payload on sign:
    const token = jwt.sign({
      userId: user.id,
      email: user.email,
      tier: normalizeTier(user.tier),
      isNewProUser: user.isNewProUser
    }, process.env.JWT_SECRET, 
    { expiresIn: "7d" })
  
  On admin tier change:
    Cannot update existing JWT — instead:
    Set requiresRefresh = true in DB
    On next API call from user, issue new token
    with updated tier embedded

STEP 3 — NORMALIZE TIER EVERYWHERE

Add this function to a shared utils file:

  // lib/normalizeTier.ts
  export const normalizeTier = (rawTier?: string) => {
    if (!rawTier) return "free"
    const t = rawTier.toLowerCase().trim()
    if (t.includes("all")) return "pro_all"
    if (
      t.includes("nba") || 
      t === "pro" || 
      t.includes("40") ||
      t.includes("25") ||
      t.includes("pro")
    ) return "pro_nba"
    return "free"
  }

  export const hasProAccess = (tier?: string) => {
    const normalized = normalizeTier(tier)
    return normalized === "pro_nba" || 
           normalized === "pro_all"
  }

Import and use normalizeTier + hasProAccess 
everywhere tier is checked — remove all 
inline tier string comparisons.

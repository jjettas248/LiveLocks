Three updates to the Today's Games strip for NCAAB Live.

---

1. NCAAB CHIPS WIDER THAN NBA CHIPS

NCAAB team names are significantly longer than NBA 
abbreviations. Adjust chip sizing for NCAAB only.

In TodaysGamesStrip component:
  Add a chipSize prop: "nba" | "ncaab"
  
  NBA chips (existing, do not change):
    min-width: 120px
    team text: abbreviation only (OKC, NY, etc.)
  
  NCAAB chips (new sizing):
    min-width: 160px
    max-width: 200px
    team text: use short display name, not full name
      e.g. "Louisiana" not "Louisiana Ragin' Cajuns"
      e.g. "James Madison" not "James Madison Dukes"
      Truncate with ellipsis if over 14 characters:
        overflow: hidden
        text-overflow: ellipsis
        white-space: nowrap
        max-width: 80px per team name

  Chip height: same as NBA (match exactly)
  Font sizes: same as NBA
  Padding: increase horizontal padding by 4px each side
    NBA: px-3 → NCAAB: px-4

  Pass chipSize to TodaysGamesStrip:
    NBA tab:   <TodaysGamesStrip chipSize="nba" ... />
    NCAAB tab: <TodaysGamesStrip chipSize="ncaab" ... />

2. STRIP GROUPED BY TIP-OFF TIME DROPDOWNS

When there are many NCAAB games, the strip organizes 
chips into collapsible tip-off time groups instead of 
a single horizontal scroll row.

LAYOUT:
  Replace single horizontal scroll row with 
  stacked time-group sections:

  Each section:
    Header row (clickable, toggles chips below):
      Left: "[TIME] CT" zinc-400 text-xs font-semibold uppercase
      Center: horizontal divider line zinc-800
      Right: chip count "[N] games" zinc-500 text-xs
             chevron icon zinc-500 (rotates on open)
    
    Chips row (collapsible):
      Horizontal scrolling row of chips for that time slot
      Same chip style as before
      scroll-behavior: smooth
      Hide scrollbar (scrollbar-hide class)
      Show scroll fade gradient on right edge if overflow

  Default open/closed state:
    Live time groups: open by default
    Upcoming time groups within 2 hours: open by default
    Later time groups: closed by default
    Past/Final time groups: closed by default

  Toggle state per group:
    const [openGroups, setOpenGroups] = useState({})
    
    // Initialize with live + soon groups open
    useEffect(() => {
      const now = Date.now()
      const initial = {}
      timeGroups.forEach(group => {
        const tipoff = new Date(group.tipoffTime).getTime()
        const diffHours = (tipoff - now) / 3600000
        const hasLive = group.games.some(g => g.status === "live")
        initial[group.key] = hasLive || diffHours <= 2
      })
      setOpenGroups(initial)
    }, [timeGroups])
    
    const toggleGroup = (key) => {
      setOpenGroups(prev => ({ ...prev, [key]: !prev[key] }))
    }

  Collapse animation:
    max-height 0 → auto, 250ms ease
    opacity 0 → 1, 200ms ease
    overflow: hidden when collapsed

  Live group header special styling:
    Left: green pulsing dot + "[TIME] CT" in green-400
    Count badge: green pill "[N] Live" instead of gray

  Final group header:
    "[TIME] CT" in zinc-600 (more muted, deprioritized)
    Chips in that group are also muted

GROUPING LOGIC:
  const groupGamesByTipoff = (games) => {
    const groups = {}
    games.forEach(game => {
      const time = formatTipoffKey(game.tipoff_time)
      // e.g. "7:00 PM" → use as group key
      if (!groups[time]) {
        groups[time] = {
          key: time,
          label: time + " CT",
          tipoffTime: game.tipoff_datetime,
          games: []
        }
      }
      groups[time].games.push(game)
    })
    // Sort groups chronologically
    return Object.values(groups).sort((a, b) => 
      new Date(a.tipoffTime) - new Date(b.tipoffTime)
    )
  }

3. GAME COUNT BADGE ON TODAY'S GAMES HEADER

Show total game count next to "TODAY'S GAMES" label.

HEADER STRUCTURE:
  <div class="strip-header">
    
    Left:
      green pulsing dot (animate-ping)
      "TODAY'S GAMES" text-xs font-semibold 
       uppercase tracking-wider zinc-400
      
      Count badge (right of label):
        bg: rgba(255,255,255,0.06)
        border: rgba(255,255,255,0.1)
        text: #a1a1aa text-xs font-mono
        rounded-full px-2 py-0.5
        Format: "· [N]"
        e.g. "TODAY'S GAMES  · 24"
        Updates reactively as games load
    
    Right:
      Live sub-count (if any games are live):
        green dot + "[N] Live" in green-400 text-xs
        hidden when liveCount === 0
      
      Refresh button (existing, keep as-is)

  </div>

COUNT VALUES:
  Total count = games.length (all games on slate)
  Live count = games.filter(g => g.status === "live").length
  
  Both update reactively on each data refresh.
  No animation on count change — instant swap.

  When all games are final:
    Total count stays (shows how many games were today)
    Live count badge disappears
    "TODAY'S GAMES" label changes to "TODAY'S SLATE · FINAL"
    Text color shifts to zinc-500

  When slate is empty (no games today):
    Count badge hidden entirely
    Strip shows empty state message instead of groups

Both changes apply globally to all probability

1. DYNAMIC CERTAINTY MULTIPLIER

The certainty multiplier must scale based on how 
much of the game has elapsed. More time played = 
more data = higher certainty = stronger lean.

FIND every location where calculateEngineProb 
or any probability calculation uses a fixed 
multiplier (e.g. * 3 or * 4).
Replace ALL of them with the dynamic version below.

DYNAMIC MULTIPLIER FUNCTION:

  const getDynamicMultiplier = (game) => {
    
    const period = game.status?.period ?? 1
    const clock = game.status?.displayClock ?? "20:00"
    const sport = game.sport ?? "ncaab"
    
    // Parse clock to minutes remaining in current period
    const [mins, secs] = clock.split(":").map(Number)
    const minutesRemainingInPeriod = 
      mins + (secs / 60)
    
    // Sport-specific period lengths
    const periodLength = sport === "nba" ? 12 : 20
    
    // Total game minutes
    const totalPeriods = sport === "nba" ? 4 : 2
    const totalGameMinutes = periodLength * totalPeriods
    
    // Minutes elapsed this period
    const elapsedThisPeriod = 
      periodLength - minutesRemainingInPeriod
    
    // Total minutes elapsed in game
    const completedPeriods = period - 1
    const totalElapsed = 
      (completedPeriods * periodLength) + elapsedThisPeriod
    
    // Progress through game: 0.0 (start) → 1.0 (end)
    const gameProgress = Math.min(
      Math.max(totalElapsed / totalGameMinutes, 0), 
      1
    )
    
    // Multiplier scale:
    // Game start (0%):   3.0 — low certainty
    // Quarter/half way:  5.0 — moderate certainty
    // Late game (75%+):  8.0 — high certainty
    // Final minutes:     12.0 — very high certainty
    
    if (gameProgress < 0.10) return 3.0   // early game
    if (gameProgress < 0.25) return 4.0   // first quarter
    if (gameProgress < 0.50) return 5.0   // first half
    if (gameProgress < 0.65) return 6.0   // mid game
    if (gameProgress < 0.75) return 7.0   // third quarter
    if (gameProgress < 0.85) return 8.0   // late game
    if (gameProgress < 0.92) return 10.0  // final minutes
    return 12.0                            // last 2 minutes
  }

2. EXTREME VALUE LIMITING TO AVOID BAD DATA

Replace the basic 2-98 clamp with a smarter 
limit system that accounts for game state.

The allowed probability range expands as the 
game progresses — extreme values are only valid 
late in games.

PROGRESSIVE LIMIT FUNCTION:

  const getProgressiveLimits = (game) => {
    
    const multiplier = getDynamicMultiplier(game)
    // Reuse game progress from multiplier calc
    
    const period = game.status?.period ?? 1
    const clock = game.status?.displayClock ?? "20:00"
    const sport = game.sport ?? "ncaab"
    const periodLength = sport === "nba" ? 12 : 20
    const totalPeriods = sport === "nba" ? 4 : 2
    const totalGameMinutes = periodLength * totalPeriods
    const [mins, secs] = clock.split(":").map(Number)
    const minutesRemainingInPeriod = mins + (secs / 60)
    const elapsedThisPeriod = 
      periodLength - minutesRemainingInPeriod
    const totalElapsed = 
      ((period - 1) * periodLength) + elapsedThisPeriod
    const gameProgress = Math.min(
      Math.max(totalElapsed / totalGameMinutes, 0), 1
    )
    
    // Allowed range expands with game progress:
    // 0-10% elapsed:   30–70  (very tight — low data)
    // 10-25% elapsed:  20–80
    // 25-50% elapsed:  10–90
    // 50-75% elapsed:   5–95
    // 75-90% elapsed:   3–97
    // 90%+ elapsed:     1–99  (near end — allow extremes)
    
    if (gameProgress < 0.10) return { min: 30, max: 70 }
    if (gameProgress < 0.25) return { min: 20, max: 80 }
    if (gameProgress < 0.50) return { min: 10, max: 90 }
    if (gameProgress < 0.75) return { min:  5, max: 95 }
    if (gameProgress < 0.90) return { min:  3, max: 97 }
    return { min: 1, max: 99 }
  }

APPLY PROGRESSIVE LIMITS:

  const limitedEngineProb = (rawProb, game) => {
    const { min, max } = getProgressiveLimits(game)
    const clamped = Math.min(Math.max(rawProb, min), max)
    
    // Log if value was clamped significantly
    if (Math.abs(rawProb - clamped) > 10) {
      console.warn(
        `[ENGINE] Probability clamped: ` +
        `raw=${rawProb.toFixed(1)}% → ` +
        `limited=${clamped.toFixed(1)}% | ` +
        `game=${game.id} | ` +
        `progress=${
          ((getProgressiveLimits(game).max - 70) / 29 * 100)
          .toFixed(0)
        }% elapsed`
      )
    }
    
    return parseFloat(clamped.toFixed(1))
  }

CONSOLE WARNING FORMAT:
  Every time a value is clamped by more than 10pp:
    [ENGINE] Probability clamped: raw=0.0% → 
    limited=30.0% | game=401234567 | 8% elapsed
  
  This surfaces bad data from ESPN or calculation 
  errors without breaking the UI.
  
  Warning threshold: only log if |raw - clamped| > 10
  Do not log small adjustments — only significant ones.

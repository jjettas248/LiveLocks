2. STAT GRID — ANIMATED VALUE TRANSITIONS

When engine probabilities update after a refresh,
stat grid values animate to their new numbers.

Applies to ALL refresh types:
  normal (<10pp): values swap instantly, no animation
  significant (>=10pp): values count up/down
  direction-change: values count up/down + color shifts

ANIMATED FIELDS (count-up/down on significant+ shifts):
  - ENGINE OVER% 
  - ENGINE UNDER%
  - STET PROJ (away projected final)
  - EKU PROJ (home projected final)
  - Spread cover probability percentage

Non-animated fields (always instant swap):
  - FULL GAME TOTAL line
  - SPREAD value
  - Current line

COUNT-UP/DOWN ANIMATION COMPONENT:
  function AnimatedNumber({ 
    value, 
    duration = 600, 
    decimals = 1,
    suffix = "",
    color
  }) {
    const [display, setDisplay] = useState(value)
    const prevValue = useRef(value)
    const frameRef = useRef(null)
    
    useEffect(() => {
      if (prevValue.current === value) return
      
      const start = prevValue.current
      const end = value
      const startTime = performance.now()
      
      const animate = (currentTime) => {
        const elapsed = currentTime - startTime
        const progress = Math.min(elapsed / duration, 1)
        
        // Ease out cubic
        const eased = 1 - Math.pow(1 - progress, 3)
        const current = start + (end - start) * eased
        
        setDisplay(parseFloat(current.toFixed(decimals)))
        
        if (progress < 1) {
          frameRef.current = requestAnimationFrame(animate)
        } else {
          prevValue.current = value
        }
      }
      
      frameRef.current = requestAnimationFrame(animate)
      return () => cancelAnimationFrame(frameRef.current)
    }, [value, duration, decimals])
    
    return (
      <span style={{ color }}>
        {display}{suffix}
      </span>
    )
  }

DIRECTION-CHANGE ADDITIONAL BEHAVIOR:
  On top of counting animation, ENGINE OVER% and 
  ENGINE UNDER% rows get a color transition:

  If flipping Over → Under:
    ENGINE OVER% color transitions: 
      #00d4aa → #ef4444 (600ms, matches count-down animation)
    ENGINE UNDER% color transitions:
      #ef4444 → #00d4aa (600ms, matches count-up animation)
  
  If flipping Under → Over:
    Reverse the above transitions

  Use CSS transition on color property:
    transition: color 600ms ease

  This means the number counts AND the color shifts 
  simultaneously — visually communicates the flip clearly

GAUGE SYNC:
  Gauge arc animation (already 500ms ease from prior prompt)
  should start at the same time as stat grid animations
  so all elements move together cohesively.
  
  Trigger all animations from the same 
  triggerFlash("direction-change") call:
    1. Start gauge arc transition
    2. Start stat grid AnimatedNumber updates
    3. Start verdict row fade out/in
    4. Start card border keyframe
    5. Fire amber toast
  
  All initiated synchronously in one event handler —
  no staggered setTimeout chains that could cause 
  visual desync between elements.

PERFORMANCE GUARD:
  AnimatedNumber uses requestAnimationFrame — 
  cancel on unmount (already handled by cleanup return).
  
  If multiple rapid refreshes fire before animation completes:
    Cancel previous animation frame
    Start new animation from current display value
    (not from original start value)
    
    Update start = display (current mid-animation value)
    not start = prevValue.current
    
    This prevents jarring jumps if refresh fires 
    mid-animation during a fast-moving game.

IMPORTANT MODEL RULES

1. HITS / TB MODEL
Base projection should combine:
- live contact quality today
- expected remaining PA / AB
- batter season skill
- pitch count fatigue
- pitch-type matchup
- weather / park
- bullpen downgrade risk
- lineup slot / order context

2. LIVE CONTACT QUALITY SCORE
Build a live contact quality score from:
- exit velo
- launch angle
- hit distance
- hard-hit classification
- barrel proxy
- quality of previous ABs today

3. TWO-AB RULE
If currentAB >= 2:
- activate full live-form adjustment
If currentAB == 1:
- only activate early explosive-contact mode if EV / LA / distance cross elite thresholds
Else:
- no live-form boost

4. LINEUP / ORDER COLLAPSE
Create a lineup context score based on:
- batting order slot
- hitters ahead getting on base
- where the pitcher is weakest in the lineup
- whether the order is turning over soon
- whether the lineup section around the hitter is collapsing or creating opportunity

5. STRIKEOUT LOGIC
For batter K:
- swing and miss
- called strike / whiff indicators if available
- pitcher pitch-type whiff fit
- expected remaining PA
- pitcher vs bullpen K environment

For pitcher K:
- pitch count
- times through order
- manager leash
- opponent lineup K pockets
- pitch-type dominance

6. HR MODEL
Treat HR as experimental.
Use:
- elite EV
- ideal launch angle band
- park / weather carry
- pitcher HR tendency
- strict thresholds
Do not trigger many alerts here.

7. HRR MODEL
Treat HRR as experimental.
Model separately:
- expected additional hits
- expected additional runs
- expected additional RBIs
Then sum.
Must include lineup context and runners-on-base opportunity proxies.

8. MODIFIER CAPS
Do not allow all modifiers to stack freely.
Structure every market as:
final_projection =
  base_projection
  + capped_live_form_adjustment
  + capped_pitcher_adjustment
  + capped_pitch_type_adjustment
  + capped_weather_park_adjustment
  + capped_lineup_adjustment
  + capped_bullpen_adjustment

Add hard caps to prevent inflated projections.

9. PROBABILITY
Use calibrated probabilities, not inflated theoretical outputs.
Build raw + calibrated probability.
Avoid frequent fake 90%+ probabilities.

- ROI by bucket

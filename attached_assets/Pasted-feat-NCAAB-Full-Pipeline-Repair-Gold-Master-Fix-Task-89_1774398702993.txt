feat: NCAAB Full Pipeline Repair (Gold Master Fix) - Task #89

Repairs the NCAAB live game pipeline so live games can still produce valid
engineInput → engineOutput → plays even when sportsbook total lines are unavailable,
by deriving a full-game total fallback and preventing validation cascade failures.

Changes to server/ncaabEngine.ts:
- Added optional liveTotalSource?: "book" | "derived" field to NCAABGameInput interface

Changes to server/ncaabService.ts:

Step 1 - Full game total derivation:
- After the existing -110 odds fallback block, inserted a pace-based total derivation block
- When total == null and the game is live, derives projected total from current score and elapsed seconds
- Clamps projected total to [100, 200], sets liveTotalSource = "derived", and logs [NCAAB DERIVED]

Step 2 - Full 6-market validation loop:
- Replaced the partial validation loop with independent validation for:
  full_total, full_spread, h1_total, h1_spread, h2_total, h2_spread
- H1/H2 failures do not cascade into full-game market invalidation
- Only null/non-finite line values invalidate a market; missing odds alone do not
- Changed h1TotalLine, h1SpreadLine, h2TotalLine, h2SpreadLine from const to let for mutability

Step 3 - Engine input extension:
- Added liveTotalSource to engineInput construction

Step 4 - Pre-engine trace replacement:
- Replaced the partial trace log with full 6-market trace logging including all in-scope odds fields and liveTotalSource

Step 5 - Hard failure detection:
- After runNCAABEngine(), logs [NCAAB FAILURE] if a live game produces zero available markets
- Corrected spec bug: markets is a Record, so availability detection uses Object.values(...).some(...)

Step 6 - SGO key trace:
- Added [NCAAB SGO TRACE H1 total] and [NCAAB SGO TRACE H2 total] logs in extractSGO1HLines()
- No lookup behavior changes; existing break statements preserved

Implementation note:
- h1UnderOddsAmerican and h2UnderOddsAmerican were not present in the current service scope, so those null assignments were omitted. This is acceptable if H1/H2 totals are intentionally modeled with only the currently scoped odds fields in this pipeline.
